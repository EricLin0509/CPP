# 动态内存简介

动态内存是C++中用于管理内存的机制，它允许程序在运行时动态地分配和释放内存

动态内存的使用使得程序能够更好地适应不同的内存需求，而不需要在编译时预先分配内存

通常动态内存存储在操作系统的堆区

## 语法

使用 `new` 关键字来分配内存，使用 `delete` 关键字来释放内存

### 单值分配

```cpp
类型 *变量名 = new 类型;
delete 变量名;

变量名 = NULL; // 推荐释放后置为NULL，避免悬空指针 (dangling pointer)
```

### 数组分配

```cpp
类型 *变量名 = new 类型[数组大小];
delete[] 变量名;

变量名 = NULL; // 推荐释放后置为NULL，避免悬空指针 (dangling pointer)
```

## 示例

### 初始化

假设我们有一个整数数组，我们希望在运行时动态分配内存

```cpp
int *ptr = new int[5]; // 分配一个大小为5的整数数组
```
- 动态内存通常用于**不知道数组长度**时的情况，这里只是作为示例

### 赋值与使用

使用解引用运算符 `*` 来赋值

```cpp
for (int i = 0; i < 5; i++)
{
    ptr[i] = (i + 1) * 2; // 赋值
}

for (int i = 0; i < 5; i++)
{
    std::cout << ptr[i] << " "; // 使用
}
std::cout << "\n";
```

### 释放内存

相比于栈，堆区内存需要用户手动释放

否则出现内存泄漏

- 内存泄漏：程序在运行时分配了内存，但没有释放，导致内存无法被回收，从而占用更多的内存

```cpp
delete[] ptr;
ptr = NULL;
```

### 内存泄漏

```cpp
while (true)
{
    int *ptr = new int; // 永远分配内存，但没有释放
}
```

这样会导致内存泄漏，程序会占用越来越多的内存，直到触发系统 OOM (Out of Memory) 机制，程序被杀死

有以下情况会导致内存泄漏

- 没有显式释放内存

```cpp
int *ptr = new int;
// 没有显式释放内存
```

- 分配内存时没有把返回的地址赋值给指针

```cpp
malloc(sizeof(int)); // 分配内存，但没有把返回的地址赋值给指针
```
