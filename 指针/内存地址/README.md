# 内存地址

内存地址是计算机中用于标识存储单元的唯一标识符

每个存储单元都有一个唯一的内存地址，可以通过这个地址访问和修改存储的数据

在C++中，内存地址可以通过取值符 `&` 来获取

## 示例

假设现在有一个变量 `int a = 10;`

那么这个 `a` 变量一般都会存储在内存中的某个位置，通过反汇编可以发现这样一段代码

```asm
LEA RAX, [RSP - 4] ; 给变量 `a` 提供内存空间
MOV DWORD PTR [RAX], 10 ; 将值 `10` 存储到变量 `a` 的内存地址中
```

- `LEA`: Load Effective Address，只做地址计算，不访问内存
- `RAX`: 一个通用寄存器
- `RSP - 4`: 计算变量 `a` 的内存地址，`RSP` 是栈指针，`- 4` 表示从栈指针减去4个字节，即4个字节的内存空间
    - 在绝大多数操作系统，栈空间是从高地址到低地址增长的
- `MOV DWORD PTR [RAX], 10`: 将值 `10` 存储到变量 `a` 的内存地址中
    - `MOV`: Move, 移动数据
    - `DWORD PTR`: Pointer to 32-bit value, 指向32位值的指针
        - 1 QWORD = 2 DWORD = 4 WORD = 16 BYTE = 64 bits
    - `[RAX]`: `RAX` 里面的数值指向的内存地址 (即 `RSP - 4` 的内存地址)
- 小声BB一句：这里使用的是 Intel 写法，x86-64 汇编还有 AT&T 写法

在C++通过取值符 `&` 可以获取变量的内存地址

```cpp
int a = 10;
std::cout << "Address of variable a: " << &a << std::endl; // 输出变量 `a` 的内存地址
```

此时输出一个十六进制数就是 `a` 变量的内存地址

```
Address of variable a: 0x7fffffffd654
```

- **注意**：这里得到的**不是物理内存地址**，而是**虚拟内存地址**
    - 原因是操作系统MMU (内存管理单元) 会做内存地址的转换
